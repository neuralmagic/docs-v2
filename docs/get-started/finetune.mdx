# Sparse Finetuning with LLMs

This guide focuses on improving the performance of large language models (LLMs) after pruning by applying finetuning techniques and then quantizing them for efficient inference.
You'll learn how to:

- <b>Recover Accuracy:</b> Mitigate potential accuracy loss from aggressive pruning by finetuning sparsified models.
- <b>Apply Finetuning and Distillation:</b> Leverage knowledge from the original model to refine the sparse model during finetuning.
- <b>Adapt to Domains:</b> Finetune LLMs to excel in particular domains or use cases.

## Prerequisites

- <b>Training Environment:</b> A system that meets the minimum hardware and software requirements as outlined in the [Install Guide](./install/#prerequisites).
- <b>SparseML LLM Installation:</b> An environment with DeepSparse for LLMs installed as outlined in the [Install Guide](./install#llms---causal-language-modeling).
- <b>Background:</b> Familiarity with Generative AI and working with large language models is recommended.

## Sparse Finetuning a Llama Model

We'll use a pre-trained, unoptimized Llama chat model from the SparseZoo trained for instruction tuning utilizing the Open Platypus dataset.
The model is referenced by the following SparseZoo stub:
```text
zoo:llama2-7b-open_platypus_orca_llama2_pretrain-base
```

For additional models, explore the [Generative AI models in the SparseZoo](https://sparsezoo.neuralmagic.com/?modelSet=generative_ai).

### One Shot Plus Sparse Finetuning

Applying pruning in one shot to an LLM can result in drops in accuracy at higher sparsity levels.
To mitigate this, SparseML can apply a recipe that includes pruning and finetuning.
Finetuning the model after pruning will help to recover any lost accuracy and improve the model's performance at higher sparsity levels.
As in the one-shot guide, we'll apply the SparseGPT algorithm and the `compress` command in SparseML to a trained model.

The code below demonstrates applying one-shot pruning, followed by finetuning with distillation, and finally, one-shot quantization to the Llama model utilizing a recipe:

<Tabs>
    <TabItem value="python" label="Python" default>

        ```python
        from sparseml.transformers import (
            SparseAutoModelForCausalLM, SparseAutoTokenizer, load_dataset, compress
        )

        model = SparseAutoModelForCausalLM.from_pretrained(
            "zoo:llama2-7b-open_platypus_orca_llama2_pretrain-base"
        )
        tokenizer = SparseAutoTokenizer.from_pretrained(
            "zoo:llama2-7b-open_platypus_orca_llama2_pretrain-base"
        )
        dataset = load_dataset("garage-bAInd/Open-Platypus")

        recipe = """
        pruning_stage:
            run_type: oneshot
            modifiers:
                - SparseGPTModifier
                    sparsity: 0.5
                    quantize: False
                    targets: [model.layers.0, model.layers.1, model.layers.2, model.layers.3, model.layers.4, model.layers.5, model.layers.6, model.layers.7, model.layers.8, model.layers.9, model.layers.10, model.layers.11, model.layers.12, model.layers.13, model.layers.14, model.layers.15, model.layers.16, model.layers.17, model.layers.18, model.layers.19, model.layers.20, model.layers.21, model.layers.22, model.layers.23, model.layers.24, model.layers.25, model.layers.26, model.layers.27, model.layers.28, model.layers.29, model.layers.30, model.layers.31, lm_head]

        finetune_stage:
            run_type: train
            modifiers:
                - ConstantPruningModifier:
                    targets: ["re:.*q_proj.weight", "re:.*k_proj.weight", "re:.*v_proj.weight", "re:.*o_proj.weight", "re:.*gate_proj.weight", "re:.*up_proj.weight", "re:.*down_proj.weight"]
                    start: 0
                - OutputDistillationModifier:
                    targets: ["re:model.layers.\\d+$"]
                    comparison: "square_head"
                    start: 0

        quantization_stage:
            run_type: oneshot
            modifiers:
                - SparseGPTModifier
                    quantize: True
                    targets: [model.layers.0, model.layers.1, model.layers.2, model.layers.3, model.layers.4, model.layers.5, model.layers.6, model.layers.7, model.layers.8, model.layers.9, model.layers.10, model.layers.11, model.layers.12, model.layers.13, model.layers.14, model.layers.15, model.layers.16, model.layers.17, model.layers.18, model.layers.19, model.layers.20, model.layers.21, model.layers.22, model.layers.23, model.layers.24, model.layers.25, model.layers.26, model.layers.27, model.layers.28, model.layers.29, model.layers.30, model.layers.31, lm_head]
        """

        compress(
            model=model,
            tokenizer=tokenizer,
            data=dataset,
            recipe=recipe,
            output_dir="./finetune-example"
        )
        ```

    </TabItem>
    <TabItem value="bash" label="Bash">

        ```bash
        echo "

            run_type: oneshot
            modifiers:
                - SparseGPTModifier

        modifiers:

                targets: ["re:.*q_proj.weight", "re:.*k_proj.weight", "re:.*v_proj.weight", "re:.*o_proj.weight", "re:.*gate_proj.weight", "re:.*up_proj.weight", "re:.*down_proj.weight"]
                start: 0
            - OutputDistillationModifier:
                targets: ["re:model.layers.\\d+$"]
                comparison: "square_head"
                start: 0

        quantization_stage:
            run_type: oneshot
            modifiers:
                - SparseGPTModifier
                    quantize: True
                    targets: [model.layers.0, model.layers.1, model.layers.2, model.layers.3, model.layers.4, model.layers.5, model.layers.6, model.layers.7, model.layers.8, model.layers.9, model.layers.10, model.layers.11, model.layers.12, model.layers.13, model.layers.14, model.layers.15, model.layers.16, model.layers.17, model.layers.18, model.layers.19, model.layers.20, model.layers.21, model.layers.22, model.layers.23, model.layers.24, model.layers.25, model.layers.26, model.layers.27, model.layers.28, model.layers.29, model.layers.30, model.layers.31, lm_head]
        " > finetune.yaml

        sparseml.transformers.apply \
            --model "zoo:llama2-7b-open_platypus_orca_llama2_pretrain-base" \
            --data "garage-bAInd/Open-Platypus" \
            --recipe ./finetune.yaml \
            --output_dir "./finetune-example"
        ```

    </TabItem>
</Tabs>

After running the above code, the model is pruned to 70% sparsity, fine-tuned, and quantized, resulting in a smaller model ready for efficient inference.

After sparsifying the model, it is ready for evaluation and deployment.
To test the model's generation capabilities, we can use the following code to generate text utilizing PyTorch:

```python
from sparseml.transformers import SparseAutoModelForCausalLM, SparseAutoTokenizer

model_path = "./one-shot-example"
model = SparseAutoModelForCausalLM.from_pretrained(model_path)
tokenizer = SparseAutoTokenizer.from_pretrained(model_path)
inputs = tokenizer(["Large language models are"], return_tensors="pt")
generated_ids = model.generate(**inputs)
outputs = tokenizer.batch_decode(generated_ids, skip_special_tokens=True)
print(outputs)
```

The above code, however, does not leverage the sparsity within the model for efficient inference.
To do so, we need to export the model to ONNX to be ready for efficient inference on CPUs with DeepSparse.
SparseML provides a simple export command to do so:

<Tabs>
    <TabItem value="python" label="Python" default>

        ```python
        from sparseml import export

        export(
            "./finetune-example",
            task="text-generation",
            sequence_length=1024,
            target_path="./exported"
        )
        ```

    </TabItem>
    <TabItem value="bash" label="Bash">

        ```bash
        sparseml.export \
            "./finetue-example" \
            --task "text-generation" \
            --sequence_length 1024 \
            --target_path "./exported"
        ```

    </TabItem>
</Tabs>

The exported model located at `./exported` can now be used for efficient inference with DeepSparse.
To do so, sub in the exported model within the previous [Getting Started - Deploy](./deploy) guide for your desired deployment method.
